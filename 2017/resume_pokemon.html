Auto-generated Steam Product FAQs

V1: Wrote a standlaone tool in C# to improve efficiency of shipping FAQs and quicktexts (pre-written snippets used by support agents) for newly released games. The tool also minimized human errors such as typos, incorrect HTML markup, and weird code generation from WYSIWYG. A support tech would launch the tool, fill out the form, and click "generate". This would create two documents - an HTML file to be pasted into the FAQ Editor and a plan text file that was automatically inserted into our quick response tool.

V2: Integrated Steam Store API into FAQ system. Instead of creating an FAQ for each product, the FAQ get generated on demand when a user hits the page. This is accomplished by cURLing the Steam store API and caching the results in memcache for 24 hours. This also allowed for more features such as system requirements, supported languages, Early Access messaging, etc. Information is always current as partners update the game via the Steam partner site.

Duplicate Ticket Experiment

Observed that as support ticket queue times increase, customers submitted additional tickets on the same issue ("duplicate tickets"). This creates multiple issues - it skews ticket metrics and generates more work for the support agents to correctly merge each user's extra tickets. Working with one of Valve's experimental psychologists, ran an experiment to see if we can curb this behavior. With a small change to support ticket creation UI, we reduced duplicate ticket creation by 65%, removing 4,900 tickets per month in total ticket volume (~196 man hours).
